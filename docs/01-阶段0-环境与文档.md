# 阶段 0：环境与文档（M0）

## 1. 阶段目标与范围

- **目标**：统一开发环境、产出落地方案文档骨架、约定项目目录与仓库规范，为后续开发提供可复现环境与文档基线。
- **范围**：不涉及业务代码实现，仅环境、文档与仓库结构。
- **前置依赖**：无。

## 2. 里程碑明细

| 编号 | 里程碑 | 交付物/验收 | 细化说明 |
|------|--------|------------|----------|
| M0.1 | 开发环境统一 | 文档：Python 版本、Node 版本（若用 Electron）、依赖管理方式 | 在 `docs/` 或 README 中写明：Python 3.x 建议版本（如 3.10+）；若 GUI 选 Electron 则注明 Node 版本；依赖管理采用 venv + requirements.txt 或 poetry，二选一并写清命令（创建环境、安装依赖）。 |
| M0.2 | 落地方案文档初版 | 完成 1～10 章骨架，关键决策写入文档 | 对应总览中的「方案文档建议结构」：项目概述、功能范围、技术架构、浏览器自动化方案、数据模型与存储、规则与 AI 筛选、私信发送与风控、GUI 设计要点、风险与合规、里程碑与排期。每章可为简要提纲，关键决策（平台、账号、筛选、AI、风控）必须写清。可放在本文档体系（00～07）或单独 `docs/plan-overview.md`。 |
| M0.3 | 项目目录与仓库规范 | 目录结构、README、.gitignore | 建立根目录下：`docs/`（方案与阶段文档）、`src/`（业务代码）、`config/`（示例配置）、`data/`（本地数据与 DB，可加入 .gitignore）；README 含项目简介、环境要求、安装与运行步骤（占位亦可）；.gitignore 含 venv、__pycache__、.env、data/*.db、敏感配置等。 |

## 3. 技术要点与实现建议

- **Python 版本**：建议 3.10+，便于使用类型注解与 match 等语法；与 Playwright、PyQt 等兼容性良好。
- **依赖管理**：若团队无强偏好，推荐 `venv + requirements.txt`，便于 CI 与文档说明；若希望锁版本与脚本统一，可用 poetry。
- **文档骨架**：先以「标题 + 要点列表」形式填满 10 章，后续各阶段文档（01～07）可视为对其中第 4～8 章的细化与更新。

## 4. 验收标准

- [ ] 新成员按文档能在本地创建环境并安装依赖（无需猜测版本与命令）。
- [ ] 方案文档 10 章均有对应内容，关键决策可追溯。
- [ ] 项目根目录结构清晰，README 与 .gitignore 已就绪，无敏感信息被纳入版本库。

## 5. 产出物清单

- 环境与依赖说明（README 或 `docs/环境与依赖.md`）。
- 落地方案文档初版（10 章骨架 + 关键决策）。
- 项目目录结构、README、.gitignore。

## 6. 与后续阶段的关系

- M0 完成后即可启动 M1（浏览器自动化）；M1 实施过程中可回头补充「浏览器自动化方案」章节的细节。
